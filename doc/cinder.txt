*cinder.txt*    cinder.nvim

cinder.nvim is a proof-of-concept Neovim plugin for running background agent
tasks.

Current provider split:
        Ask uses a pi RPC adapter.
        Do defaults to a fake provider scaffold, but can be overridden with
        configured profiles.

COMMANDS                                                *cinder-commands*

:Cinder
        Open or reuse the composer buffer.

:Cinder --profile {name}
        Open or reset the composer buffer onto a named profile.

:Cinder send
        Submit the draft from the composer buffer.

:Cinder do [--profile {name|default}] {prompt}
        Start a background Do run for the current editing context.

:Cinder new [--profile {name|default}]
        Start a fresh composer session, optionally on a named profile.

:Cinder runs
        Open a scratch buffer listing in-memory runs.

:Cinder kill {run_id}
        Cancel a run by id.

:Cinder doctor
        Open a scratch report with provider/config validation.

CONFIGURATION                                        *cinder-configuration*

>lua
  require("cinder").setup({
    provider = "pi",
    model = nil,

    profiles = {
      fake = { provider = "fake", model = "fake-do" },
      fast = { provider = "pi", model = "gpt-5-mini" },
    },

    ask = {
      profile = nil,
    },

    inline = {
      profile = "fake",
    },

    providers = {
      pi = {
        cmd = "pi",
        args = { "--mode", "rpc", "--no-session" },
      },
      fake = {
        interval_ms = 30,
        total_ticks = 4,
      },
    },
  })
<

Use `default` as the profile name to clear an override and fall back to the
top-level `provider` and `model`.
